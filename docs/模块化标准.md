# Sumi-Kara 模块化开发标准

## 1. 模块化原则

### 1.1 核心设计理念

- **高内聚低耦合**：每个模块只负责单一功能域
- **接口驱动**：模块间通过明确定义的 API 通信
- **可插拔**：模块支持独立开发、测试、部署
- **向后兼容**：新增功能不破坏现有接口

---

## 2. 模块分类

### 2.1 前端模块 (React)

```
src/modules/
├── {module-name}/
│   ├── index.tsx              # 模块入口
│   ├── components/            # 模块私有组件
│   ├── hooks/                 # 自定义 Hooks
│   ├── types.ts               # TypeScript 类型定义
│   ├── api.ts                 # API 调用封装
│   └── __tests__/             # 单元测试
```

### 2.2 后端模块 (Python)

```
backend/modules/
├── {module_name}/
│   ├── __init__.py            # 模块入口
│   ├── router.py              # FastAPI 路由
│   ├── service.py             # 业务逻辑
│   ├── schemas.py             # Pydantic 数据模型
│   ├── utils.py               # 工具函数
│   └── tests/                 # 单元测试
```

## 3. 模块开发规范

### 3.1 前端模块接口规范

```typescript
// 模块必须导出以下接口
export interface ModuleDefinition {
  name: string;           // 模块名称
  version: string;        // 模块版本
  init: () => Promise<void>;
  destroy: () => void;
}

// 示例：歌词模块
export const LyricsModule: ModuleDefinition = {
  name: 'lyrics',
  version: '1.0.0',
  init: async () => { /* 初始化逻辑 */ },
  destroy: () => { /* 清理逻辑 */ }
}
```

### 3.2 后端模块接口规范

```python
# 模块路由注册
from fastapi import APIRouter

router = APIRouter(prefix="/api/{module}", tags=["{module}"])

@router.get("/health")
async def health_check():
    return {"status": "ok"}

# 模块注册表
MODULES = {
    "lyrics": "backend.modules.lyrics.router",
    "alignment": "backend.modules.alignment.router",
    "style": "backend.modules.style.router",
}
```

### 3.3 模块配置规范

```typescript
// config/modules.ts
export interface ModuleConfig {
  enabled: boolean;
  endpoint: string;
  settings?: Record<string, any>;
}

export const MODULE_CONFIG: Record<string, ModuleConfig> = {
  lyrics: {
    enabled: true,
    endpoint: 'http://localhost:8000/api/lyrics',
  },
  alignment: {
    enabled: true,
    endpoint: 'http://localhost:8000/api/alignment',
  },
  style: {
    enabled: false,  // 未实现时禁用
    endpoint: 'http://localhost:8000/api/style',
  },
}
```

## 4. 模块通信机制

### 4.1 事件总线 (Event Bus)

```typescript
// events/types.ts
export type ModuleEvent =
  | { type: 'LYRICS_LOADED'; payload: LyricsData }
  | { type: 'ALIGNMENT_COMPLETE'; payload: TimelineData }
  | { type: 'STYLE_APPLIED'; payload: StyleConfig };

// 使用示例
eventBus.subscribe('LYRICS_LOADED', (data) => {
  timeline.updateLyrics(data);
});
```

### 4.2 数据流

```
用户操作 → React Component → Module API →
FastAPI Service → FFmpeg →
WebSocket Progress → React Component → UI 更新
```

## 5. 新增模块流程

### 5.1 创建模块模板

```bash
# 前端模块
mkdir -p src/modules/{module-name}/{components,hooks,__tests__}

# 后端模块
mkdir -p backend/modules/{module_name}/{tests,utils}
```

### 5.2 模块注册

1. 在 `src/modules/index.ts` 导出新模块
2. 在 `backend/main.py` 注册 API 路由
3. 在 `config/modules.ts` 添加配置
4. 更新本文档

### 5.3 模块测试

```bash
# 前端测试
npm run test:unit -- src/modules/{module-name}

# 后端测试
pytest backend/modules/{module_name}/tests
```

## 6. 模块版本管理

| 版本号 | 说明 |
|--------|------|
| MAJOR | 不兼容的 API 变更 |
| MINOR | 向后兼容的功能新增 |
| PATCH | 向后兼容的问题修复 |

### 6.1 模块变更日志

每个模块应维护 `CHANGELOG.md`：

```markdown
## [1.0.0] - 2026-03-04

### Added
- 初始版本
- 基础功能实现

### Changed
- 无

### Fixed
- 无
```

## 8. 模块开发检查清单

开发新模块前请确认：

- [ ] 模块职责明确，不越界

- [ ] 无循环依赖
